# 成都市车友会管理系统

## 项目简介

成都市车友会管理系统是一个面向车友会日常运营与管理的后台工程，主要用于承载活动管理、会员申请、报名审核、权益维护、轮播内容维护以及系统基础配置等业务能力。

该工程当前作为成都市车友会后端管理系统使用，既提供管理端页面能力，也承载面向业务模块的服务实现，便于统一维护车友会相关数据与运营流程。

## 主要功能

1. 活动管理：维护车友会活动信息、活动状态、活动名额与相关配置。
2. 报名管理：处理活动报名、审核流转与参与人数统计。
3. 会员申请：维护会员申请资料、审核状态与业务处理过程。
4. 会员信息管理：维护车友基础资料与后台管理数据。
5. 权益管理：维护车友会福利、权益内容与展示信息。
6. 轮播与公告管理：维护门户展示所需的轮播图、公告与相关内容。
7. 系统配置：维护管理系统运行所需的基础参数与界面配置。

## 工程结构

- `ruoyi-admin`：可运行的后台管理应用，包含管理端页面与控制器入口。
- `ruoyi-motorclub`：成都市车友会核心业务模块，包含活动、报名、会员、权益等业务实现。
- `ruoyi-framework`：系统安全、权限、AOP、MyBatis 等通用运行支撑。
- `ruoyi-system`：系统级用户、角色、菜单、部门等基础能力。
- `ruoyi-common`：公共工具、基础实体与通用封装。
- `ruoyi-quartz`：定时任务能力。
- `ruoyi-generator`：代码生成能力。

## 本地启动

### 构建工程

```bash
mvn clean package -DskipTests
```

### 启动后台系统

```bash
mvn -pl ruoyi-admin -am spring-boot:run
```

### 打包指定运行模块

```bash
mvn -pl ruoyi-admin -am clean package -DskipTests
```

## 适用场景

本工程适用于成都市车友会后台管理、活动运营、会员审核、内容维护以及日常系统配置等业务场景，也可作为车友会相关管理能力的后端基础工程持续扩展。
